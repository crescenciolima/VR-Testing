# Test Case – Performance Efficiency

## Objective

Evaluate the time behavior (loading time, frame rate, and input latency) and resource utilization (CPU, GPU, memory, and battery) during metaverse usage on a mid-range Android mobile device (4 GB RAM, octa-core processor).

---

## Steps

1. Launch the metaverse through a web browser on the mobile device after clearing the browser cache and closing all background applications.
2. Measure the time from clicking the "Start" button until the 3D environment is fully loaded and the character Maria Felipa appears.
3. Navigate through the environment for 15 minutes, interacting with NPCs and completing at least one minigame and one quiz.
4. During usage, monitor:
a. Average frames per second (FPS) using a performance monitoring application.
b. Latency between user commands and system response (e.g., button click → interface reaction).
c. CPU, GPU, memory consumption, and battery percentage before and after the test.

---

## Expected Result

- Initial loading time is ≤ 10 seconds.
- Stable frame rate of ≥ 30 FPS throughout navigation.
- Input latency of ≤ 150 ms during interactions with menus, NPCs, and minigames.
- CPU and GPU utilization remains within a moderate usage range (≤ 70%).
- RAM usage remains ≤ 2.5 GB.
- Battery consumption is ≤ 8% after 15 minutes of use.
- No crashes, freezes, or noticeable performance degradation occur during the test period.
