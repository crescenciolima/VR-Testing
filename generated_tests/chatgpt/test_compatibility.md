# Test Case – Compatibility

## Objective

Verify that the metaverse can be used simultaneously with other applications without compromising performance (coexistence) and that user progress is correctly synchronized across devices (interoperability between mobile, browser, and VR platforms).

---

## Steps

1. Access the metaverse on an Android smartphone, complete a minigame, and save the progress.
2. Switch between the metaverse and another resource-intensive application (e.g., YouTube running in the background) while navigating the metaverse for at least 10 minutes.
3. Then, access the metaverse using the same user account through a web browser on a computer and verify that the minigame progress has been synchronized correctly.
4. Repeat the access using a VR headset and validate again that the saved state is up to date.

---

## Expected Result

- The metaverse continues to function normally even with another application active in the background, without crashes, significant FPS drops, or interruptions to user interactions (coexistence).
- Progress completed on the smartphone (finished minigame) is correctly displayed when accessing the metaverse through both the web browser and the VR headset, with no data loss or synchronization delays (interoperability).
- All devices can consistently access the same cloud-stored profile and progress data.
