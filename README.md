# VR-Testing

**Can LLMs Write Your Tests? Evaluating Automated ISO/IEC 25010-Based Test Generation for VR Metaverse Environments**

A research project that investigates the capability of Large Language Models (LLMs) to automatically generate software quality test cases for immersive VR/metaverse applications, evaluated against the ISO/IEC 25010 quality model.

---

## Overview

This project evaluates how well state-of-the-art LLMs can generate meaningful, structured test cases for a real-world educational metaverse: an immersive 3D environment that recreates the **Battle of Itaparica** from a decolonial perspective. The metaverse features historical characters (such as Maria Felipa as a guide), minigames, quizzes, NPC interactions, and cloud-based progress saving — running on browsers, mobile devices, and VR headsets via the **Spatial.io / Unity** platform.

The study uses standardized prompts to query each LLM and collects the generated test cases for comparative analysis.

---

## Research Question

> *To what extent can Large Language Models autonomously generate valid and useful software test cases, aligned with the ISO/IEC 25010 quality model, for a VR metaverse application?*

## Quality Attributes Under Study (ISO/IEC 25010)

Five quality attributes are evaluated, each with targeted sub-attributes:

| Attribute | Sub-attributes |
|---|---|
| **Reliability** | Maturity, Fault Tolerance, Recoverability |
| **Performance Efficiency** | Time Behavior (FPS, latency, load time), Resource Utilization (CPU/GPU/memory/battery) |
| **Usability** | Appropriateness Recognizability, Learnability, Operability |
| **Security** | Confidentiality, Integrity, Non-repudiation |
| **Compatibility** | Coexistence, Interoperability (cross-platform sync) |

## Methodology

1. **Prompt design** — One structured prompt per ISO/IEC 25010 attribute is crafted, providing the LLM with: the quality attribute and sub-attributes, the target application description, and the required output format (`Objective / Steps / Expected Result`).
2. **Data collection** — Each prompt is submitted to each LLM. Responses are saved as Markdown files under `generated_tests/<llm>/`.
3. **Evaluation** — Generated test cases are assessed for completeness, correctness, alignment with ISO/IEC 25010, and practical applicability in a VR/metaverse context. The assessment is automated through an **LLM-as-a-Judge** (see below), which scores each test case on a common set of quality criteria and produces an aggregate report.

---

## LLM-as-a-Judge

The test cases collected in `generated_tests/` are evaluated automatically by an **LLM-as-a-Judge**: an impartial evaluator, expert in ISO/IEC 25010 software quality, packaged as a [Claude skill](https://docs.claude.com/en/docs/claude-code/skills). The skill lives in [`LLM-as-a-Judge/vr-testing-judge.skill`](LLM-as-a-Judge/vr-testing-judge.skill) (a zip archive containing a single `SKILL.md`).

Given a batch of test cases and the ISO attribute/sub-attributes they target, the judge scores **every** test case on four criteria:

| Criterion | Question |
|---|---|
| **Coverage** | Does the test address the declared ISO attribute and its sub-characteristics? |
| **Executability** | Can the test be run on the current prototype without missing dependencies? |
| **Clarity** | Are the objective, steps, and expected result unambiguous? |
| **Novelty** | Does the test add value beyond standard regression scenarios? |

Each criterion is rated on a five-point ordinal scale — Very High (10), High (8), Medium (6), Low (4), Very Low (2). The **per-test score** is the mean of the four criteria; the **aggregate score** is the mean across all tests. The output is a single Markdown report with per-test evaluation tables, justifications, and an aggregate summary.

### Installing the skill in Claude

The skill is distributed as a `.skill` bundle and runs in [Claude Code](https://docs.claude.com/en/docs/claude-code/overview) (or the Claude desktop app). To install it:

1. **Locate the skills directory.** Claude reads skills from `~/.claude/skills/` (personal, available everywhere) or, for project-scoped use, from `.claude/skills/` inside this repository.
2. **Unpack the bundle into that directory.** The archive already contains a top-level `vr-testing-judge/` folder, so unzip preserves the expected layout:

   ```bash
   # personal install (available in every project)
   unzip LLM-as-a-Judge/vr-testing-judge.skill -d ~/.claude/skills/

   # — or — project-scoped install (only this repo)
   unzip LLM-as-a-Judge/vr-testing-judge.skill -d .claude/skills/
   ```

   This yields `~/.claude/skills/vr-testing-judge/SKILL.md` (or `.claude/skills/vr-testing-judge/SKILL.md`).
3. **Restart Claude Code** (or start a new session) so the skill is discovered. Confirm it loaded by running `/skills` — `vr-testing-judge` should appear in the list.

### Using the judge

Open Claude Code in the project and ask it to evaluate a batch, e.g.:

> *Run the LLM-as-a-Judge on `generated_tests/chatgpt/test_security.md` for the Security attribute (Confidentiality, Integrity, Non-repudiation).*

Trigger phrases such as *"evaluate tests"*, *"judge test cases"*, *"score my tests"*, or *"run the judge"* also activate the skill. It asks for the target ISO attribute/sub-attributes and the test cases (pasted inline or as `.md` files), then returns the scored report.
